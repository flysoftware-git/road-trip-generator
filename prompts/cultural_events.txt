Synthesize the following web search results into cultural event entries for a road trip itinerary.

Destination:      {destination_name}
Travel dates:     {dates}
Destination type: {destination_type}
  (values: national_park | small_town | resort_town | city)

Search results (Bing API, raw):
{bing_results}

════════════════════════════════════════════════════════
CORE RULE: HONESTY OVER COMPLETENESS
════════════════════════════════════════════════════════

If the search results do not contain substantive events within or immediately
adjacent to the travel date range, DO NOT invent events. Return Format B
(honest fallback) below. A traveler arriving to find a fabricated festival
is a far worse outcome than an itinerary that honestly describes a quiet scene.

════════════════════════════════════════════════════════
DECISION TREE — follow in order
════════════════════════════════════════════════════════

Step 1 — Scan search results for events meeting ALL of these criteria:
  a) Occurs during or within 7 days of the travel dates above
  b) Is a specific named event — not a "things to do" listicle or general guide
  c) Is a performance, exhibition, festival, market, or cultural program
     NOT a restaurant listing, hotel deal, or standard park attraction

Step 2 — Count qualifying events from Step 1.

Step 3 — If count >= 2: return FORMAT A (events found)
          If count < 2:  return FORMAT B (honest fallback)
                         This includes ALL national_park destinations
                         and most small_town destinations.

Do not treat:
  - "annual summer festival" results as evidence of an October event
  - Events from prior years as current without explicit recurrence confirmation
  - General visitor guides as event listings

════════════════════════════════════════════════════════
QUALITY RULES for FORMAT A
════════════════════════════════════════════════════════

- Maximum 5 events — do not pad
- Source preference order: official event site > local arts council >
  CVB/tourism board > local newspaper > aggregator (Eventbrite, Facebook last)
- url field: verbatim from Bing results only — no construction, no guessing
- venue_address: only if street address appears literally in search results
- Nearby-city events: always note distance from destination

════════════════════════════════════════════════════════
STRICT PROHIBITIONS
════════════════════════════════════════════════════════

- NEVER invent event names, venues, or dates
- NEVER construct or infer URLs not present verbatim in search results
- NEVER treat prior-year events as current without recurrence confirmation
- NEVER use: stunning, breathtaking, iconic, vibrant, lively, charming,
  bustling, thriving
